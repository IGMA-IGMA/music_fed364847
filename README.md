# 🎵 Music Catalog Platform

[![GitHub top language](https://img.shields.io/github/languages/top/IGMA-IGMA/music_fed364847)](https://github.com/IGMA-IGMA/music_fed364847/search?l=html)
[![Python Version](https://img.shields.io/badge/python-3.8%2B-blue)](https://www.python.org/)
[![Django Version](https://img.shields.io/badge/Django-6.0.3-green)](https://www.djangoproject.com/)
[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)

**Music Catalog Platform** — веб-приложение, построенное на Django, предназначенное для создания и управления каталогом музыкальных исполнителей и их композиций. Платформа предоставляет удобный интерфейс для просмотра информации об артистах, прослушивания треков и выполнения глобального поиска.

---

## ✨ Основные возможности

- **Каталог исполнителей** — страницы артистов с биографией, фотографией и количеством слушателей.
- **Библиотека песен** — детальная информация о каждой композиции, включая название, длительность и количество прослушиваний.
- **Аудиоплеер** — прослушивание треков в формате MP3 непосредственно на сайте.
- **Глобальный поиск** — быстрый поиск по всем артистам и песням с поддержкой частичного совпадения.
- **Регистрация пользователей** — форма для регистрации новых пользователей.
- **Автоматизация коммитов** — утилита для автоматического анализа изменений и создания коммитов в Git-репозитории.

---

## 🛠 Технологии

- **Backend**: Python, Django, Django ORM
- **База данных**: SQLite (стандартная), PostgreSQL (опционально через `psycopg2`)
- **Фронтенд**: HTML, CSS
- **Коммуникации**: gRPC
- **Управление зависимостями**: Poetry
- **Инструменты**: GitPython, pipreqs

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/IGMA-IGMA/music_fed364847.git
cd music_fed364847
````

### 2. Настройка виртуального окружения

```bash
python -m venv venv
source venv/bin/activate        # Для Linux/macOS
venv\Scripts\activate           # Для Windows
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

_Альтернативно, если используется Poetry:_

```bash
poetry install
```

### 4. Применение миграций базы данных

```bash
python artists/manage.py migrate
```

### 5. Создание суперпользователя (для доступа к админ-панели)

```bash
python artists/manage.py createsuperuser
```

### 6. Запуск сервера разработки

```bash
python artists/manage.py runserver
```

После запуска приложение будет доступно по адресу: [http://127.0.0.1:8000/](http://127.0.0.1:8000/)

## 🔍 Использование

### Просмотр каталога

- **Главная страница** — отображает список всех артистов.
- **Страница артиста** — подробная информация об исполнителе и список его песен.

### Поиск

Используйте строку поиска в интерфейсе для нахождения артистов или песен по названию. Результаты сортируются по релевантности.

### Администрирование

- Перейдите в `/admin` и войдите под учётной записью суперпользователя.
- Управляйте артистами, песнями и другими данными через стандартную административную панель Django.

## 🛠 Дополнительные утилиты

### Автоматический коммит (`autocommit.py`)

Скрипт для автоматического добавления всех изменений, создания коммита с осмысленным сообщением и отправки изменений в удалённый репозиторий.

```bash

```

Содержит реализацию быстрой сортировки (`speedSort`) для демонстрационных целей.
